避免返工和设计流程的优化

工作中，返工恐怕是最最浪费时间的原因之一。一个项目，因为前一个工作流程没做好，从而导致后面的工作白做，这种情况并不少见。

不过这也是现实，总会有一些意外的不可抗力发生，有的返工是不可避免的，但有的返工是因为工作流程或者工作方法不当而导致的，这种情况是可以通过优化流程来避免。

一、返工的重要原因

返工的原因之一在于设计输入条件没有做扎实，比如Base图频繁发生变化。

一套方案文本、初设或者施工图大概可以划分为两个部分，一部分是内容，一部分是对内容的注释。一旦内容发生了改变，注释就白做。

因此，在工作流程上，注释应该往后放。并且，注释一般是重复性的体力活，且重要程度较低，可以分配出去。

以施工图为例：

种植设计中应该在苗木规格稳定再进行植物平面布，否则空间大小不对，平面布置确定以后再进行植物标注。

竖向设计图应在明确使用的坐标系统（绝对坐标或相对坐标），以及明确与外部环境衔接处的标高和地标径流组织的方向之后再开始进行标高推算。

…

到了施工图这一步，每张图的绘制都是基于前面很多流程确定的内容为基础，稍有些不确定就要改动。作为最后的步骤，遇到返工的可能性就越大。

由此可见，设计是有许多层次的，从上位规划»>方案设计»>初步设计»>施工图设计，其实每一个层次的意义就在于发现问题，及时解决一些问题，同时敲定一些控制性的内容，避免之后的返工。

但在实际的工作中流程经常是混乱且交叉进行的，比如在施工图设计的时候，其实还有部分方案没有稳定。

并不是说必须要在上一步情况的稳定之后才能开展之后的工作，而是说，要明白有哪些没有稳定，哪些内容可能会翻车重做，以及这种清楚这种风险是可以承担的前提下，就可以开展下一步工作。

避免返工的关键是优化工序，以及稳定设计输入条件。

二、设计流程优化

按照设计的先后顺序，做一个输入输出的清单。每一步都是上一步工序的承接，也是下一步工序的前提，按照这个流程做，能够减小返工和重复无效工作的可能。

仅列出目前有限的理解，后面待完善。

Input 规划阶段的输入条件

  1. 甲方的诉求（识别并回应甲方的诉求、显性目的或隐形目的，以及赢得甲方的初步信任可能是设计中最重要的事了，跟这一点相比画图什么的都是次要的）

  2. 经济上的目的（如果甲方想要的是吸引游客，结果做了一个以功能性和生态性为主的东西，完全是没有抓住重点）

  3. 生态上的目的

  4. 功能上的目的

  5. 意向、概念、IP、卖点和流量

Output 规划阶段的输出结果

  1. 甲方明确的诉求

Input 方案阶段的输入条件

  1. 强制性规范

  2. 甲方的诉求

  3. 预算

  4. 上位规划

  5. 场地所处的人文环境

  6. 场地所处的自然环境context（现场调查的目的）

  7. 场地的交通环境、现状高程、植被和建筑体（可以通过对场地进行建模来了解）

  8. 设计理念、概念和主旨（有的方案其实并没这些，只有炫目的手法）

  9. 设计风格和手法（古典、现代、抽象、非线性、参数化）

Output 方案阶段的输出结果

  1. 大致的总平面图（主要的园路布置，主要节点的形态，主要建筑的造型）

  2. 大致的竖向方案（主要控制点的高程、空间感塑造）

  3. 大致的种植方案（主要苗木及规格的确定）

  4. 大致的建筑方案

Input 初步设计阶段的输入条件

  1. 总平面图（主要的园路布置，主要节点的形态，主要建筑的造型）

  2. 大致的竖向方案（主要控制点的高程、空间感塑造）

  3. 大致的种植方案（主要苗木及规格的确定）

  4. 大致的建筑方案

Output 初步设计阶段的输出结果

  1. 比较明确的总平面图

  2. 比较明确的竖向方案

  3. 比较明确的种植方案

  4. 比较明确的建筑方案

  5. 比较明确的结构、给排水、电方案

Input 施工图设计阶段的输入条件

  1. 比较明确的总平面图

  2. 比较明确的竖向方案

  3. 比较明确的种植方案

  4. 比较明确的建筑方案

  5. 比较明确的结构、给排水、电方案

Output 施工图设计阶段的输出结果

  1. 明确的总平面图

  2. 明确的竖向布置

  3. 明确的种植布置

  4. 明确的建筑布置

  5. 明确的结构、给排水、电布置

以上只是大概列举了一下，每一项条件和结果还可以细分，有空再细写。

下次在工作的时候，可以对照检查下设计的输入条件是否成熟和稳定，判断清是否具备条件后再开始行动。
